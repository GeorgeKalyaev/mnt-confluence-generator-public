# Решение проблем с Docker

## Docker не найден в командной строке

Если вы видите ошибку "docker не является внутренней или внешней командой":

### Решение 1: Запустите Docker Desktop

1. Найдите "Docker Desktop" в меню Пуск
2. Запустите приложение
3. Дождитесь, пока Docker Desktop полностью запустится (иконка в трее перестанет мигать)
4. Попробуйте снова выполнить команду `docker --version`

### Решение 2: Перезапустите терминал

После установки Docker Desktop нужно:
1. Закрыть все окна командной строки/PowerShell
2. Открыть новое окно
3. Попробовать команду `docker --version`

### Решение 3: Проверка установки

1. Откройте Docker Desktop
2. В настройках (Settings) → General
3. Убедитесь, что включено "Use the WSL 2 based engine" (если используете WSL)
4. Или отключите, если WSL не установлен

### Решение 4: Ручной запуск через Docker Desktop

Если команды не работают, можно использовать GUI Docker Desktop:
1. Откройте Docker Desktop
2. Перейдите на вкладку "Containers"
3. Нажмите "Run" или "Add"
4. Введите образ: `atlassian/confluence-server:latest`
5. Настройте порты: 8090:8090
6. Запустите контейнер

## Альтернатива: Использовать готовый скрипт

После того как Docker заработает, используйте готовые скрипты:
- `start-confluence.bat` (для cmd)
- `start-confluence.ps1` (для PowerShell)

## Проверка что Docker работает

Выполните в терминале:
```bash
docker --version
docker ps
```

Если обе команды работают - Docker готов к использованию!
