# Офлайн зависимости

Проект полностью автономен и не требует подключения к интернету во время работы.

## Внешние зависимости, которые были локализованы

### ✅ Bootstrap 5.3.0
- **CSS**: `app/static/css/bootstrap.min.css`
- **JS**: `app/static/js/vendor/bootstrap.bundle.min.js`

Эти файлы скачаны локально и подключены через `url_for()` в шаблонах.

### ✅ Шрифты
Используются системные шрифты (не требуют интернета):
- `-apple-system` (macOS/iOS)
- `BlinkMacSystemFont` (Chrome на macOS)
- `Segoe UI` (Windows)
- `Roboto` (Android)
- `Helvetica Neue`, `Arial`, `sans-serif` (fallback)

Google Fonts удалены из шаблонов.

## Python зависимости

Все зависимости из `requirements.txt` **зафиксированы с точными версиями** для воспроизводимости сборок.

### Скачивание зависимостей для офлайн-установки

Для полностью офлайн установки в закрытой корпоративной сети можно скачать все зависимости локально:

#### Вариант 1: Скачать в папку packages/ (рекомендуется для включения в репозиторий)

```bash
# Скачать все зависимости в папку packages/
pip download -r requirements.txt -d ./packages

# Установка с локальной папки (без интернета)
pip install --no-index --find-links ./packages -r requirements.txt
```

После скачивания папка `packages/` будет содержать все `.whl` и `.tar.gz` файлы пакетов. Эту папку можно включить в репозиторий или передать отдельно.

#### Вариант 2: Создать локальное зеркало PyPI

Если в компании есть внутренний PyPI mirror/proxy, можно настроить pip для его использования.

### Важность версий

✅ **В `requirements.txt` все версии зафиксированы** - это гарантирует:
- Воспроизводимые сборки
- Совместимость между зависимостями
- Предсказуемое поведение приложения
- Возможность офлайн установки без неожиданных обновлений

## Проверка офлайн-режима

Для проверки что проект не зависит от интернета:

1. Отключите интернет на машине разработки
2. Запустите приложение: `python -m uvicorn app:app --reload`
3. Откройте в браузере: http://localhost:8000
4. Проверьте что все стили и скрипты загружаются корректно

Все ресурсы должны загружаться локально из `app/static/`.
