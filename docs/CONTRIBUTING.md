# Руководство по внесению вклада

Спасибо за интерес к проекту МНТ Confluence Generator!

## Как внести вклад

### Сообщение об ошибках

Если вы нашли ошибку:
1. Проверьте, не была ли она уже зарегистрирована в Issues
2. Создайте новый Issue с подробным описанием:
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Версия Python, PostgreSQL, ОС

### Предложение новых функций

1. Создайте Issue с описанием функции
2. Объясните, зачем нужна эта функция
3. Предложите способ реализации (если есть идеи)

### Pull Requests

1. Fork репозитория
2. Создайте ветку для вашей функции (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Убедитесь, что код соответствует стилю проекта
5. Закоммитьте изменения (`git commit -m 'Add amazing feature'`)
6. Запушьте в ветку (`git push origin feature/amazing-feature`)
7. Откройте Pull Request

## Стандарты кода

- Используйте Python 3.10+
- Следуйте PEP 8
- Добавляйте docstrings для функций и классов
- Пишите понятные имена переменных и функций
- Комментируйте сложную логику

## Структура проекта

```
app/
├── config.py          # Конфигурация приложения
├── main.py            # FastAPI приложение и роуты
├── database.py        # Подключение к БД
├── db_operations.py   # Операции с БД
├── models.py          # Модели данных
├── confluence.py      # Интеграция с Confluence
├── render.py          # Рендеринг в Confluence формат
├── templates/         # HTML шаблоны
└── static/            # Статические файлы (CSS, JS)
```

## Тестирование

Перед отправкой PR убедитесь, что:
- Приложение запускается без ошибок
- Все существующие функции работают
- Новый код не ломает существующий функционал

## Вопросы?

Если у вас есть вопросы, создайте Issue с меткой `question`.
